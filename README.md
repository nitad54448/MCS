# Scherrmann Lab Microreactor DoE Suite

Welcome to the **Scherrmann Lab Microreactor DoE (Design of Experiments) Suite**. This repository contains two interconnected HTML/JavaScript applications designed to plan, execute, and analyze continuous-flow microreactor experiments. 

These tools operate entirely client-side in the browser, requiring no external server or backend.

---

## 1: Microreactor Experiment Planner (`plan_MCS_v12.html`)

The Experiment Planner generates highly specific, mathematically rigorous JSON experiment plans based on user-defined chemical and physical parameters. It calculates precise pump flow rates, slug tracking, collection times, and syringe refill schedules.
This JSON file is used in our lab in a specific system programmed in Labview.

### Features
* **Dynamic Flow Calculations:** Automatically computes flow rates required to hit target Residence Times (RT) based on a defined Reactor Volume ($V_R$).
* **Dead Volume Tracking:** Accounts for pre-reactor mixer volume ($V_1$) and post-reactor tubing to collector volume ($V_2$).
* **Automated Step Sequencing:** Generates precise timings for Step 1 (Slug Formation) and Step 2 (Slug Transit, Reaction, Collection, and Flush).
* **Refill Management:** Calculates syringe depletion and automatically injects "Refill Pauses" into the experiment timeline when total required volume exceeds syringe capacity.
* **Diffusion Accounting:** Uses a custom Diffusion Coefficient ($C_d$) to collect leading and trailing shoulders of the reaction slug.

### Mathematical & DoE Methods

#### Design of Experiments (DoE)
The planner supports multiple statistical designs to explore the parameter space (Residence Time, B/A Molar Ratio, Active Flow Fraction, Temperature):

1.  **Full Factorial (2-level):** Tests all combinations of the minimum and maximum boundaries. For $k$ factors, this generates $2^k$ experiments. Great for identifying primary effects and linear interactions.
2.  **Box-Behnken Design (3-level):** An independent, quadratic design that requires fewer runs than a 3-level full factorial. It creates a matrix where factors are tested at their minimum (-1), center (0), and maximum (+1) levels. *Note: Strict implementation requires exactly 3 varying factors.*
3.  **Central Composite Design (CCD):** Builds upon a factorial design by adding "axial" (star) points and "center" points. Allows for estimating the curvature of the response surface.
    * **Standard CCD:** $\alpha = (2^k)^{0.25}$ to maintain rotatability.
    * **Simple CCD:** $\alpha = 1$ (Face-centered).

#### Chemical Flow Mathematics
To formulate the required mixture from stock solutions $A$ and $B$, the tool solves the following system to find volumetric flow rates $Q_A$ and $Q_B$:

Given an Active Flow Fraction (AFF), the target active flow is $Q_{active} = Q_{total} \times \text{AFF}$.
The volumetric flow distribution depends on the molar ratio target $R_{target}$ and stock concentrations $[A]$ and $[B]$:

$$Q_A = \frac{Q_{active}}{1 + R_{target} \left( \frac{[A]}{[B]} \right)}$$
$$Q_B = Q_{active} - Q_A$$

### Output
The application outputs a heavily structured `.json` file containing all global configurations, targeted set-points, and an array of calculated step-by-step instructions for the physical pumps.

## 2: Experiment Data Analyzer (`analyse_MCS_12.html`)

The Data Analyzer ingests the JSON file generated by the Planner. After executing the physical experiments, we input the measured Yield and Selectivity directly into this tool to model the response surfaces and predict mathematical optimums.

### Features
* **Adaptive Modeling:** Automatically detects the underlying DoE matrix type (2-level vs 3+ level) and adjusts its mathematical regression to prevent matrix singularity.
* **Dual-Objective Optimization:** Allows users to set weight percentages for Yield vs. Selectivity to calculate a customized "Weighted Score" for each experiment.
* **2D Contour Visualization:** Generates color-mapped topographical representations of interacting factor pairs.
* **Printable Reports:** Built-in CSS `@media print` rules format the page into a clean, presentation-ready PDF report.

### Mathematical Methods

#### Ordinary Least Squares (OLS) Regression
The tool utilizes matrix algebra to fit a Response Surface Model (RSM) over the experimental data. 

Let $X$ be the design matrix of inputs and $Y$ be the matrix of measured responses. The coefficients matrix $B$ is found using the normal equation:
$$B = (X^T X)^{-1} X^T Y$$

* **Matrix Inversion:** The inverse $(X^T X)^{-1}$ is calculated manually using **Gauss-Jordan elimination**.
* **Adaptive Terms:** * If the data contains at least 3 distinct levels for the interacting factors, a **Full Quadratic Model** is built (6 terms): 
        $$Response = \beta_0 + \beta_1x + \beta_2y + \beta_3xy + \beta_4x^2 + \beta_5y^2$$
    * If the data contains only 2 levels (e.g., from a Full Factorial plan), the squared terms ($x^2, y^2$) cannot be computed due to zero degrees of freedom for curvature. The model automatically downgrades to a **Linear Interaction Model** (4 terms):
        $$Response = \beta_0 + \beta_1x + \beta_2y + \beta_3xy$$

#### Optimum Prediction via Grid Search
To find the predicted theoretical maximum for the contour plots, the tool runs a high-resolution grid search. It divides the bounded area into a 30x30 grid ($900$ discrete evaluation points) and calculates the predicted $Z$ value (Response) at each $(x,y)$ coordinate using the derived $\beta$ coefficients. The maximum derived value is capped at $100\%$ (for Yield/Selectivity logic) and plotted on the canvas as a star (★).

### Usage
1.  Open the file in a modern browser.
2.  Drag and drop the JSON file generated by the Planner into the upload zone.
3.  Fill in the measured Yield and Selectivity results in the generated table. *You can type decimals with a comma or dot.*
4.  If enough variance is detected ($\ge 4$ distinct points for linear planes, $\ge 6$ points for curves), the 2D contour plots and predicted optimums will automatically render.
